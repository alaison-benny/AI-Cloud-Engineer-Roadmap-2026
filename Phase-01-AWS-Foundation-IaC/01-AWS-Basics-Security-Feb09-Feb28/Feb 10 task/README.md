# AWS S3 Foundations: Hands-on Practice

ഈ പ്രോജക്റ്റിൽ, ഒരു AWS EC2 Linux ഇൻസ്റ്റൻസിൽ നിന്ന് **AWS CLI** ഉപയോഗിച്ച് എങ്ങനെ ഒരു S3 ബക്കറ്റ് നിർമ്മിക്കാമെന്നും ഫയലുകൾ കൈകാര്യം ചെയ്യാമെന്നും വിവരിക്കുന്നു. ഒരു 'Compliance-Led AI Cloud Engineer' എന്ന നിലയിൽ ഡാറ്റാ സ്റ്റോറേജ് കൈകാര്യം ചെയ്യുന്നതിന്റെ ആദ്യ ഘട്ടമാണിത്.

## ലക്ഷ്യം (Objective)

* AWS CLI കോൺഫിഗർ ചെയ്യുക.
* ഒരു പുതിയ S3 ബക്കറ്റ് നിർമ്മിക്കുക.
* EC2-ൽ നിന്ന് S3-ലേക്ക് ഫയലുകൾ അപ്‌ലോഡ് ചെയ്യുക.

---

## ഘട്ടങ്ങൾ (Steps)

### 1. AWS CLI കോൺഫിഗർ ചെയ്യുക

ആദ്യമായി നിങ്ങളുടെ EC2 സിസ്റ്റത്തിന് S3-ലേക്ക് ആക്സസ് ഉണ്ടെന്ന് ഉറപ്പാക്കണം. ടെർമിനലിൽ താഴെ പറയുന്ന കമാൻഡ് ടൈപ്പ് ചെയ്യുക:

```bash
aws configure

```

> **ശ്രദ്ധിക്കുക:** ഇവിടെ നിങ്ങളുടെ `Access Key`, `Secret Key`, `Region` (ഉദാഹരണത്തിന്: `eu-central-1`) എന്നിവ നൽകുക.

### 2. ഒരു പുതിയ S3 ബക്കറ്റ് നിർമ്മിക്കുക

S3 ബക്കറ്റിന്റെ പേര് ലോകമെമ്പാടും യുണീക് (Unique) ആയിരിക്കണം. താഴെ പറയുന്ന കമാൻഡ് ഉപയോഗിച്ച് ബക്കറ്റ് ക്രിയേറ്റ് ചെയ്യാം:

```bash
aws s3 mb s3://my-compliance-bucket-2026

```

*(കുറിപ്പ്: `my-compliance-bucket-2026` എന്നതിന് പകരം നിങ്ങളുടെ ഇഷ്ടമുള്ള പേര് നൽകാം.)*

### 3. ഒരു ഫയൽ നിർമ്മിച്ച് അപ്‌ലോഡ് ചെയ്യുക

ആദ്യം ഒരു സാമ്പിൾ ടെക്സ്റ്റ് ഫയൽ നിർമ്മിക്കാം, ശേഷം അത് ബക്കറ്റിലേക്ക് കോപ്പി ചെയ്യാം:

```bash
# ഫയൽ നിർമ്മിക്കാൻ
echo "This is a compliance test file" > testfile.txt

# ഫയൽ അപ്‌ലോഡ് ചെയ്യാൻ
aws s3 cp testfile.txt s3://my-compliance-bucket-2026/

```

### 4. ബക്കറ്റിലെ ഫയലുകൾ പരിശോധിക്കുക

ഫയൽ വിജയകരമായി അപ്‌ലോഡ് ആയോ എന്ന് പരിശോധിക്കാൻ താഴെ പറയുന്ന കമാൻഡ് ഉപയോഗിക്കാം:

```bash
aws s3 ls s3://my-compliance-bucket-2026/

```

## പഠിച്ച കാര്യങ്ങൾ (Key Takeaways)

* **S3 (Simple Storage Service):** സ്കെയിലബിൾ ആയ ഒബ്ജക്റ്റ് സ്റ്റോറേജ്.
* **AWS CLI:** കമാൻഡ് ലൈൻ വഴി ക്ലൗഡ് റിസോഴ്സുകൾ നിയന്ത്രിക്കാനുള്ള ടൂൾ.
* **Data Security:** ബക്കറ്റ് നിർമ്മിക്കുമ്പോൾ മുതൽ ഡാറ്റാ സെക്യൂരിറ്റി ഉറപ്പാക്കേണ്ടതിന്റെ പ്രാധാന്യം.
